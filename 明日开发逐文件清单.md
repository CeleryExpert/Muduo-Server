## 6. 新建 `src/EventLoop.cc`
实现顺序：
1. 构造：`epoll_create1(EPOLL_CLOEXEC)`。
2. `setReadCallback(fd, cb)`：
   - 保存回调到 map。
   - `epoll_ctl ADD/MOD` 关注 `EPOLLIN`。
3. `removeFd(fd)`：
   - `epoll_ctl DEL`。
   - map 删除回调。
4. `loop()`：
   - `epoll_wait` 阻塞等待。
   - 遍历事件，按 fd 找回调执行。
5. `quit()`：设置 `quit_ = true`。

验收：
1. 用一个 pipe 或监听 fd 验证回调能触发。

## 10. 新建 `src/TcpServer.cc`
实现顺序：
1. `start()`：
   - `acceptor_.listen()`。
   - 给 `acceptor_.fd()` 注册 `loop_.setReadCallback(...handleNewConnection...)`。
   - 进入 `loop_.loop()`。
2. `handleNewConnection()`：
   - `accept()` 拿 `connfd`。
   - 创建 `TcpConnection`。
   - 绑定回调：
     - message 回调 -> `handleMessage`
     - close 回调 -> `handleClose`
   - 将 `connfd` 注册到 `EventLoop` 可读回调，回调里调用 `conn->handleRead()`。
   - 放入 `connections_`。
3. `handleMessage()`：
   - 直接 `conn->send(input)`（Echo）。
   - 清理 `input`（由你选择策略）。
4. `handleClose()`：
   - 从 `EventLoop` 删除 fd。
   - 从 `connections_` 删除。
   - 记录日志。
5. `stop()`：
   - `loop_.quit()`。
